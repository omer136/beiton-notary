## קריטריוני למידה (מה מחפשים)

### 1. לידים אבודים
- שיחות עם **2+ הודעות לקוח** אך **ללא פרטי קשר** (`hasContact=false`)
- שיחות שבהן הלקוח שאל שאלה ברורה (שירות/מחיר) אבל הסוכן לא ביקש טלפון/מייל
- **יעד:** פחות מ-20% מהשיחות ללא פרטי קשר

### 2. שאלות ללא מענה
- חיפוש מחרוזות כמו "לא יודע", "אבדוק", "I don't know", "let me check"
- זיהוי דפוסי שאלות חוזרות שהסוכן לא מטפל בהן טוב
- **פעולה:** להוסיף תשובות ל-system prompt

### 3. נטישה אחרי הודעה ראשונה
- שיחות עם `msgCount=1` — הלקוח שאל פעם אחת ועזב
- אם החלק הזה גדול מ-40% → ההודעה הראשונה של הסוכן לא מזמינה מספיק
- **פעולה:** לשפר את ה-greeting או את תגובת המענה הראשונה

### 4. העברה לנציג אנושי (needsHuman)
- שיחות עם `needs_human=true` — למה הלקוח ביקש אדם?
- זיהוי תחומים שהסוכן לא מכסה טוב או שדורשים עדכון מחירון
- **יעד:** פחות מ-10% מהשיחות

### 5. טעויות מחירים
- חיפוש אזכורים של מחירים שגויים (למשל 35₪ במקום 41₪, 40₪ במקום 41₪)
- חיפוש חישובי מע"מ על אגרות (שגוי!)
- **פעולה:** אם נמצא — עדכון `pricing-config.ts` ו-`agent1-system-prompt.ts` מיד

### 6. תמחור/סיכום לא מפורמט
- חיפוש שיחות שבהן פירוט מחיר הוצג כטקסט רץ במקום רשימה
- **פעולה:** לחזק את כלל הפורמט ב-system prompt

### 7. כשלים בעקרון פתיחת השיחה
- הודעה ראשונה של הסוכן שחסרה בה הצעה לעזור
- אי-בקשת שם מוקדם בשיחה
- אי-שימוש ב-`capture_lead` גם כשיש פרטים

### 8. פילוח לפי שפה
- האם שיחות באנגלית/ערבית/רוסית מובילות לפחות לידים מוצלחים?
- בעיות תרגום ב-prompt

### 9. פילוח לפי שירות
- איזה שירות מייצר הכי הרבה לידים? הכי מעט?
- איפה הסוכן "תוקע" לקוחות?

### 10. זיהוי תלונות
- חיפוש: "לא עזרת", "זה לא מה ששאלתי", "you're not helpful", מילות תסכול
- **פעולה:** תיקון מיידי

## כלי גישה — API

הנקודה מוגנת בסוד `ADMIN_SECRET` (צריך להגדיר ב-Vercel env vars).

```
GET /api/admin/chats?secret=YOUR_SECRET&<filters>
```

### פרמטרים

| פרמטר | דוגמה | תיאור |
|-------|--------|--------|
| `from` | `2026-04-01` | תאריך התחלה (כולל) |
| `to` | `2026-04-07` | תאריך סיום (כולל) |
| `language` | `he` / `en` / `ru` / `ar` / `fr` / `es` | שפת השיחה |
| `service` | `תרגום` | חיפוש חלקי בשירות |
| `status` | `פנייה` | סטטוס הליד |
| `minMsgs` | `2` | מספר הודעות מינימלי של הלקוח |
| `hasContact` | `true` | רק שיחות עם טלפון או מייל |
| `searchText` | `מחיר` | חיפוש בתוך התמלול |
| `limit` | `50` | מקסימום פריטים (ברירת מחדל 50, מקסימום 500) |

### דוגמאות שימוש

**כל השיחות השבועיות בעברית:**
```
/api/admin/chats?secret=X&language=he&from=2026-04-01&to=2026-04-07
```

**לידים אבודים (2+ הודעות אבל בלי פרטי קשר):**
```
/api/admin/chats?secret=X&minMsgs=2&hasContact=false
```

**שיחות שבהן הוזכר אפוסטיל:**
```
/api/admin/chats?secret=X&searchText=אפוסטיל
```

**שיחות שבהן הסוכן אמר "אבדוק":**
```
/api/admin/chats?secret=X&searchText=אבדוק
```

```
   - מדדי המפתח (total, with_contact, avg_msgs, needs_human)
   - 3-5 דוגמאות בולטות לכשלים
   - המלצות קונקרטיות לעדכון ה-system prompt

4. עדכן — אחרי אישור:
   - ערוך src/lib/agent1-system-prompt.ts
   - קומיט + דחיפה
   - אמת שהבאג לא חוזר בשיחות חדשות
```

## שימוש מתוך Claude Code

כשאתה (Omer) רוצה שאני אריץ ניתוח, תגיד:

- **"נתח שיחות מהשבוע"** → אריץ query עם `from` השבוע, אעבור על הקריטריונים ואחזיר דוח
- **"מצא לידים אבודים"** → query עם `minMsgs=2&hasContact=false`
- **"חפש שיחות עם אפוסטיל שבהן הסוכן טעה"** → query עם `searchText=אפוסטיל`, סקירה ידנית
- **"עדכן את הסוכן לפי מה שלמדנו"** → ערוך את system prompt לפי הממצאים

אני אשתמש ב-`curl` או `fetch` ל-endpoint מתוך הטרמינל (עם `ADMIN_SECRET` מ-`.env.local`).
