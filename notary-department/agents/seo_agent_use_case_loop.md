## מבנה עמוד תוכן

כל עמוד מקרה שימוש כולל את הסקשנים הבאים בדיוק:

### 1. H1 — כותרת ממוקדת מילת מפתח

קרא מעמודה `text_mm1xwbza` (כותרת מוצעת). אם ריקה — צור כותרת שכוללת:
- סוג השירות
- סוג המסמך
- המילה "נוטריוני" או "נוטריון"

**דוגמה:** "תרגום נוטריוני של תעודת גירושין — מדריך מלא ומחירון 2026"

### 2. מבוא — 2-3 משפטים

תיאור קצר של המקרה, למי זה רלוונטי, ולמה צריך נוטריון.

**דוגמה:**
> תעודת גירושין מבית דין רבני בישראל דורשת תרגום נוטריוני כשמגישים בקשת ויזה או הגירה למדינה אחרת.
> הנוטריון מאשר שהתרגום נאמן למקור — וזה הופך את המסמך למוכר רשמית בחו״ל.

### 3. "מתי צריך את זה?" — סיטואציות נפוצות

רשימת bullet points של מקרים שבהם אנשים צריכים את השירות הזה.

**דוגמה:**
- הגשת בקשת ויזת עבודה לקנדה / אוסטרליה / ארה״ב
- רישום נישואין חדשים בחו״ל
- הוכחת סטטוס אישי לרשויות הגירה
- הליך משפטי בחו״ל שדורש הוכחת גירושין

### 4. "מה התהליך?" — צעד אחרי צעד

קרא מ-`config/notary_services.json` את ה-`delivery_options` ו-`documents_required` של השירות, ובנה רשימה ממוספרת.

**דוגמה:**
1. שלח/י צילום של תעודת הגירושין בוואטסאפ או במייל
2. שלח/י צילום תעודת זהות
3. אנחנו מתרגמים ושולחים הצעת מחיר
4. אחרי אישור — התרגום מוכן תוך 3 ימי עבודה
5. מקבלים סריקה דיגיטלית + מקור בשליח

### 5. "כמה זה עולה?" — מחשבון מחירים

קרא מ-`config/notary_pricing_2026.json` וחשב דוגמה ספציפית למקרה.

**פורמט:**

| תוספת תרגום ע״י נוטריון | XXX ₪ |
| שכר תרגום | XXX ₪ |

> אין הבדל במחיר בין נוטריונים — התעריף אחיד בכל הארץ.

### 6. "איך מקבלים את השירות?"

קרא את `delivery_options` מ-`notary_services.json` והצג את האפשרויות הרלוונטיות.

## Schema Markup

כל עמוד חייב לכלול 3 סוגי Schema:

### 1. Service Schema

```json
{
  "@context": "https://schema.org",
  "@type": "Service",
  "name": "[שם השירות]",
  "description": "[תיאור קצר]",
  "provider": {
    "@type": "LegalService",
    "name": "משרד עורכי דין בית-און ושות׳",
    "address": {
      "@type": "PostalAddress",
      "streetAddress": "רח׳ רוטשילד 42",
      "addressLocality": "תל אביב-יפו",
      "postalCode": "6688312",
      "addressCountry": "IL"
    }
  },
  "areaServed": "IL",
  "availableLanguage": ["he", "en", "ru", "ar", "fr", "es"]
}
```

### 2. FAQ Schema

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "[שאלה]",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "[תשובה]"
      }
    }
  ]
}
```

### 3. BreadcrumbList Schema

```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "דף הבית", "item": "https://www.firm.co.il/" },
    { "@type": "ListItem", "position": 2, "name": "נוטריון", "item": "https://www.firm.co.il/notary/" },
    { "@type": "ListItem", "position": 3, "name": "[שם השירות]", "item": "https://www.firm.co.il/notary/[slug]/" },
    { "@type": "ListItem", "position": 4, "name": "[שם המקרה]", "item": "https://www.firm.co.il/notary/[slug]/[use-case-slug]/" }
  ]
}
```

## יצירה ב-6 שפות

כל עמוד נוצר ב-6 גרסאות שפה:

| שפה | URL pattern | hreflang |
|---|---|---|
| עברית | `/notary/[slug]/[uc-slug]/` | `he` |
| English | `/en/notary/[slug]/[uc-slug]/` | `en` |
| Русский | `/ru/notary/[slug]/[uc-slug]/` | `ru` |
| العربية | `/ar/notary/[slug]/[uc-slug]/` | `ar` |
| Français | `/fr/notary/[slug]/[uc-slug]/` | `fr` |
| Español | `/es/notary/[slug]/[uc-slug]/` | `es` |

**חשוב:**
- שמות שירותים בכל שפה — קרא מ-`notary_services.json` (שדות `name_he`, `name_en`, `name_fr`, וכו׳)
- שמות אופני מסירה — קרא מ-`delivery_methods` (שדות `name_fr`, `name_es`, וכו׳)
- **מונחים עבריים בסוגריים** גם בגרסאות השפה האחרות, כי הלקוח יצטרך את המונח בעברית מול הרשויות
